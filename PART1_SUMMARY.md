## 1. نطاق الجزء الأول (المنجز الآن)

| الوحدة | الوصف |
|---|---|
| **البنية الأساسية** | طبقة تخزين JSON مستقلة، سجل تدقيق (Audit Log)، مولّد أكواد تلقائي |
| **لوحة التحكم (Dashboard)** | مؤشرات حية + رسوم بيانية + أحدث الحوادث + أعلى المخاطر أولوية |
| **إدارة خطط السلامة** | CRUD كامل + نظام إصدارات تلقائي (Version Control) + اعتماد الخطط + خطط الإخلاء |
| **إدارة المخاطر (Risk Assessment)** | مصفوفة 5×5 وفق ISO 45001 + هرم التحكم بالمخاطر + متابعة إجراءات التحكم |
| **إدارة الحوادث والإصابات** | سجل كامل + تحقيق إلزامي قبل الإغلاق + حساب معدلي OSHA (التكرار والشدة) |

### أبرز القرارات الهندسية الحقيقية (وليست شكلية)
- **مصفوفة المخاطر**: `الدرجة = الاحتمالية (1-5) × الشدة (1-5)`، تُصنَّف تلقائياً: منخفض (≤4) / متوسط (≤9) / عالٍ (≤15) / حرج (>15).
- **منع إغلاق الحوادث بدون تحقيق حقيقي**: الكود يرفض تغيير حالة أي حادث إلى "مغلق" ما لم يتوفر سبب مباشر + سبب جذري + إجراء تصحيحي واحد مكتمل على الأقل (تم اختبار هذا السيناريو فعلياً).
- **معدلات OSHA/ISO 45001 الحقيقية**:
  - معدل تكرار الحوادث = (عدد الحوادث القابلة للتسجيل × 200,000) ÷ إجمالي ساعات العمل
  - معدل شدة الإصابات = (إجمالي أيام العمل الضائعة × 200,000) ÷ إجمالي ساعات العمل
- **تحكم تلقائي بحالة الخطر**: عند اكتمال جميع إجراءات التحكم الخاصة بخطر ما، تتحول حالته تلقائياً إلى "مُتحكَّم به".
- **إصدارات خطط السلامة**: أي تعديل جوهري على المحتوى (الأهداف، السياسات، الإجراءات...) يرفع رقم الإصدار تلقائياً ويُسجَّل في سجل تاريخي منفصل.

## 4. الأجزاء المتبقية (خطة التنفيذ)

### 🔲 الجزء 2/4 — التفتيشات، تصاريح العمل، معدات الوقاية الشخصية (PPE)
- **إدارة التفتيشات**: جولات تفتيش + قوائم تحقق (Checklist) قابلة للتخصيص + تسجيل ملاحظات وصور + تحديد مخالفات ومسؤول ومعالجة + إعادة تفتيش + اعتماد النتائج.
- **تصاريح العمل (Permit to Work)**: 8 أنواع تصاريح (ارتفاعات، أماكن مغلقة، لحام، كهرباء، حفر، رفع، مواد خطرة، صيانة) مع دورة موافقات (Approval Workflow) وتقييم مخاطر مرتبط بكل تصريح وتنبيه عند الانتهاء.
- **معدات الوقاية الشخصية (PPE)**: سجل مخزون كامل (خوذ، أحذية، قفازات...) مع تتبع تسليم/استبدال لكل موظف، العمر الافتراضي، وتنبيهات انتهاء الصلاحية.
- **ملفات متوقعة**: `backend/utils/hseInspections.js` (أو دمجها في `hseManagement.js` كجزء ثانٍ)، تحديثات على `server.js`، `frontend/js/hse.js`، ولوحات جديدة في `index.html`.

### 🔲 الجزء 3/4 — الطوارئ، التدريب، المواد الخطرة، المخالفات، مكافحة الحريق
- **إدارة الطوارئ**: سيناريوهات، فرق طوارئ، تدريبات إخلاء دورية، تقييم الاستجابة.
- **إدارة التدريب**: دورات سلامة، شهادات مع تواريخ تجديد، اختبارات، تقييم متدربين.
- **إدارة المواد الخطرة**: تسجيل + بطاقات بيانات السلامة (SDS) + أماكن تخزين + إجراءات طوارئ متخصصة.
- **إدارة المخالفات**: تسجيل مستقل عن الحوادث والتفتيشات + إجراءات تصحيحية + مواعيد إغلاق.
- **معدات مكافحة الحريق**: طفايات، إنذار، رش آلي + جداول صيانة وفحص دوري.

### 🔲 الجزء 4/4 — التنبيهات الذكية، التقارير، الرسوم البيانية، الذكاء الاصطناعي، التكامل والصلاحيات المتقدمة
- **التنبيهات الذكية**: تنبيهات تلقائية مجدولة (انتهاء تصاريح، صلاحية PPE، اقتراب تفتيش، ارتفاع معدل الحوادث...) — تتطلب تكامل مع نظام الجدولة/الإشعارات المركزي.
- **التقارير**: تصدير PDF/Excel/CSV/Word لكل من (الحوادث، المخاطر، التفتيش، التصاريح، الأداء، التقرير التنفيذي) — سيُبنى على نمط `equipmentReports.js` الموجود.
- **الذكاء الاصطناعي**: تحليل أنماط الحوادث، تنبؤ بالمخاطر، اقتراح إجراءات وقائية، تلخيص تقارير — على نمط `equipmentIntelligence.js`.
- **التكامل الكامل**: ربط فعلي (وليس شكلياً) مع إدارة المشاريع، الجدول الزمني، المعدات، العمال، الجودة، المستندات، المشتريات، المخازن.
- **الصلاحيات المتقدمة**: تفعيل نظام `can(token, 'safety', action)` بشكل صارم على كل route (حالياً الجزء الأول لم يُفعِّل `requirePermission` صراحة على مسارات `/api/hse/*` تجهيزاً لدمج شامل للصلاحيات في الجزء الرابع، تماماً كما فعل القسم السابع في `equipmentIntelligence.js` تحت "الصلاحيات المتقدمة").

## 5. ملاحظة تقنية مهمة لاستكمال الأجزاء القادمة

جميع الوظائف الجديدة (تفتيش، تصاريح، PPE، طوارئ، تدريب...) ستُضاف كدوال إضافية داخل **نفس ملف** `backend/utils/hseManagement.js` (أو ملفات مساعدة يستدعيها، بنفس نمط `equipmentReports.js` و`equipmentIntelligence.js` في القسم السابع)، مع توسيع كائن `module.exports` تدريجياً — دون المساس بأي دالة من الجزء الأول. كذلك ستُضاف مسارات API جديدة إلى **نفس** كائن `API_HANDLERS` في `server.js` دون حذف أو تعديل أي مسار حالي.
