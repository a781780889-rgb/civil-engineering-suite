# خطة تنفيذ القسم الثاني عشر: إدارة المخططات الهندسية (10 أجزاء)

## ✅ الجزء 1/10 — الأساس (منجَز في هذه الجلسة)
- طبقة تخزين موحّدة (`backend/data/drawings.json` + `backend/data/drawings_files/`)
- التخصصات السبعة وأنواعها الفرعية (إنشائي، معماري، كهربائي، ميكانيكي، طرق، مساحة، BIM)
- أنواع الملفات المدعومة: DWG, DXF, DGN, RVT, IFC, PDF, DWF, STEP, STL, صور
- توليد رقم مخطط تلقائي (مثال: `DRW-STR-PRJ001-0007`)
- إنشاء مخطط + رفع الملف الفعلي كإصدار v1 (checksum SHA-256 حقيقي)
- تنزيل الملف الفعلي، تعديل البيانات الوصفية، حذف منطقي (soft delete)
- لوحة التحكم الرئيسية (كل الإحصائيات المطلوبة في الصفحة الرئيسية بالمواصفة)
- سجل تدقيق (Audit Log) لكل عملية
- صلاحية جديدة `drawings` مضافة لكل الأدوار في `biz_roles.json`
- ملفات: `backend/utils/drawingManagement.js`, `backend/frontend/js/drawings.js`
- ربط: `server.js` (مسارات `/api/drawings/*`), `frontend/index.html` (nav + panels)

## 🔲 الجزء 2/10 — إدارة الإصدارات المتقدمة
- رفع إصدار جديد لمخطط موجود (v2, v3, ...)
- سجل كامل للإصدارات لكل مخطط (من قام بالتعديل، متى، ملاحظة التغيير)
- استعادة إصدار سابق (يصبح هو الحالي دون فقد التاريخ)
- تجهيز البنية لمقارنة الإصدارات (التي ستُستخدم فعلياً في الجزء 7/10)

## 🔲 الجزء 3/10 — عارض المخططات (طبقة API)
- بيانات وصفية للقياسات (مسافات/زوايا/مساحات/أطوال) المحفوظة من المستخدم على كل مخطط
- حفظ نقاط قراءة الإحداثيات
- دعم فتح/عرض عدة مخططات معاً (API لإرجاع مجموعة ملفات دفعة واحدة)
- ملاحظة: العرض الرسومي الفعلي لملفات CAD يعتمد على مكتبات عميل (frontend) خارج نطاق الـ backend؛ الجزء يوفر البيانات والقياسات المرتبطة

## 🔲 الجزء 4/10 — إدارة الطبقات (Layers)
- تخزين وصفي لطبقات كل مخطط (اسم الطبقة، مرئية/مخفية، مقفلة، الترتيب، التصنيف)
- تشغيل/إيقاف/قفل/إعادة ترتيب الطبقات عبر API
- بحث عن طبقة داخل مخطط

## 🔲 الجزء 5/10 — مراجعة المخططات
- إرسال للمراجعة → مراجعة هندسية → اعتماد/رفض/إعادة للمصمم → اعتماد نهائي
- تحديث `approval_status` تلقائياً وربطه بلوحة التحكم (حقل `total_reviews` سيُحسب هنا فعلياً)
- سجل كامل لكل خطوة مراجعة (من، متى، القرار، السبب)

## 🔲 الجزء 6/10 — التعليقات والملاحظات
- كتابة تعليق + تحديد موضعه (إحداثيات x/y) على المخطط
- الرد على التعليقات، إغلاق التعليق، تصنيف الملاحظات، متابعة حالة التنفيذ
- حقل `total_comments` في لوحة التحكم سيُحسب هنا فعلياً

## 🔲 الجزء 7/10 — مقارنة المخططات
- مقارنة إصدارين لنفس المخطط (باستخدام سجلات الإصدارات من الجزء 2/10)
- إظهار عناصر مضافة/محذوفة/معدّلة (مقارنة checksum + بيانات وصفية للطبقات/الأبعاد/النصوص المخزَّنة يدوياً أو عبر استيراد)

## 🔲 الجزء 8/10 — إدارة الاعتمادات المتعددة
- اعتماد داخلي / استشاري / عميل / مقاول / نهائي — كل مستوى له سجل مستقل
- توقيع إلكتروني + ختم زمني + سجل اعتمادات كامل

## 🔲 الجزء 9/10 — التكامل مع BIM
- تسجيل بيانات نماذج Revit/IFC/Navisworks/Civil 3D/Tekla المرتبطة بمخطط
- اكتشاف تعارضات وصفي (Clash Detection) بين عناصر مسجَّلة يدوياً/عبر استيراد
- ربط عناصر النموذج بالمخططات

## 🔲 الجزء 10/10 — التنبيهات + التقارير + الذكاء الاصطناعي + التكامل الشامل
- تنبيهات: رفع مخطط جديد، إصدار جديد، مراجعة جديدة، اعتماد، رفض، تعديل، انتهاء مهلة المراجعة، تعارض إصدارات
- تقارير احترافية (مخططات/مراجعات/إصدارات/تعليقات/اعتمادات/مشاريع + تقرير تنفيذي) مع تصدير PDF/Excel/CSV/Word
- مساعد ذكي: تحليل المخططات، اكتشاف تعارضات بين التخصصات، مقارنة تلقائية، عناصر مفقودة، مراجعة توافق الأكواد، اقتراح تحسينات، تحليل أثر التعديلات، ملخصات مراجعات، تصنيف تلقائي، تقارير تحليلية
- الربط التلقائي الكامل مع بقية الأقسام (المشاريع، المساحة، حصر الكميات، الجودة، السلامة، المستندات، الأعمال، المعدات، الجدول الزمني، التقارير) بنفس نمط `documentModuleLinks.js`
