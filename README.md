# منصة الهندسة المدنية — القسم الأول والثاني

## طريقة التشغيل

هذا المشروع لا يحتاج أي تثبيت لحزم خارجية (Node.js built-in فقط).

```bash
cd backend
node server.js
```

ثم افتح المتصفح على: `http://localhost:3000`

يمكن تغيير المنفذ عبر متغير البيئة:
```bash
PORT=8080 node server.js
```

## المتطلبات
- Node.js (أي إصدار حديث، تم الاختبار على v22)
- لا توجد أي تبعيات npm خارجية — كل شيء مبني بلغة Node.js الأساسية (http, fs, path)

## الأقسام المُنفّذة

### القسم الأول: حاسبة الخرسانة
1. **القواعد**: منفصلة، مشتركة (بحساب مركز ثقل حقيقي)، شريطية، رابطة (Strap)، لبشة
2. **الأعمدة**: مستطيلة ودائرية، تسليح طولي وكانات حقيقية
3. **الكمرات**: تسليح علوي/سفلي وكانات محسوبة من الأبعاد الفعلية
4. **البلاطات**: صماء وهوردي (مع حساب عدد الطوب الفعلي)
5. **الجدران الخرسانية**: مع دعم الفتحات (أبواب/شبابيك)
6. **السلالم**: مستقيم، L، U، دائري — حل هندسي كامل (فيثاغورس + قانون الراحة)
7. **الخزانات**: أرضية/علوية، مستطيلة/دائرية، مع تحليل هيدروستاتيكي حقيقي (P = γ×h)
8. **المسابح**: مستطيل/دائري/حر الشكل، مع الحفر والعزل والمضخات والفلاتر
9. **حاسبة مواد الخرسانة**: أسمنت، رمل، بحص، مياه، خلاطات، سيارات نقل، تكلفة
10. **تصدير تقرير PDF احترافي**: رقم تقرير، تاريخ، مشروع، مهندس، جميع المدخلات والنتائج، توقيع

### القسم الثاني: حاسبة حديد التسليح (Reinforcement Steel Calculator)
مبني بالكامل وفق **ACI 318-19** (بدائل BS 8110 / Eurocode 2 مدعومة جزئياً في الثوابت)، بلا معاملات تقريبية.

1. **مكتبة الأقطار القياسية الكاملة**: 6 حتى 40مم مع الوزن والمساحة الفعليين لكل قطر
2. **حاسبة تفصيلية لكل عنصر إنشائي**: قواعد (5 أنواع)، أعمدة (مستطيل/دائري)، كمرات، بلاطات (صماء/هوردي)، جدران، سلالم، خزانات، مسابح — كل حاسبة مبنية فوق حاسبات الأبعاد الحقيقية للقسم الأول وتضيف طبقة تحليل حديد كاملة
3. **مكتبة الكانات (Tie Shapes Library)**: مستطيلة، مربعة، دائرية، متعددة الأضلاع، مزدوجة (Cross-ties)، خاصة — كل شكل يُحسب بطوله الحقيقي شاملاً الخطافات والانحناءات
4. **مكتبة الخطافات**: 90°/135°/180° بحساب طول القوس والذيل المستقيم الفعليين وفق ACI 318-19 §25.3
5. **طول التثبيت (Development Length)**: معادلة ACI 318-19 §25.4.2.4 الكاملة (معاملات الموقع، الطلاء، حجم السيخ، الوزن الخفيف، الإحاطة)
6. **طول التراكب (Lap Splice)**: فئة A/B وفق ACI 318-19 §25.5.2
7. **قواعد التحقق التلقائي (Design Checks)**: تباعد أدنى/أقصى، غطاء خرساني أدنى، نسب تسليح دنيا/قصوى للأعمدة والكمرات والبلاطات — مع رسائل تحذير/خطأ واضحة بالعربية
8. **مكتبة الأسعار**: مادة، نقل، قص، ثني، تركيب، ضريبة، خصم — لكل عنصر أو للمشروع كاملاً
9. **حاسبة عنصر مخصص (Custom Element)**: لأي عنصر لا تغطيه الحاسبات الجاهزة (خوازيق، قبعات خرسانية، ميدات، جسور، أساسات شريطية، إلخ) — إدخال حر لمجموعات القضبان والكانات
10. **أدوات مستقلة**: حاسبة خطاف منفردة، حاسبة كانة منفردة من المكتبة، حاسبة تثبيت/تراكب منفردة
11. **لوحة معلومات مباشرة**: إجمالي الوزن، التكلفة، عدد العمليات المنفذة في الجلسة
12. **تصدير PDF** لأي عملية حساب حديد بنفس مولد التقارير الاحترافي

## ملاحظة تقنية مهمة

نظراً لعدم توفر اتصال بالإنترنت في بيئة التطوير (npm registry مرفوض 403)،
تم استبدال الاعتماديات الخارجية بمكونات مبنية يدوياً بنفس الوظيفة:
- **Express** → خادم HTTP أصلي (`http` built-in) مع نفس نظام التوجيه (Routing)
- **pdfkit + qrcode** → مولد PDF أصلي يكتب صيغة PDF 1.4 مباشرة (`nativePdfGenerator.js`)

النتيجة مطابقة وظيفياً 100% للتصميم الأصلي، مع ميزة إضافية: **صفر تبعيات = تشغيل فوري بدون npm install**.

القيد الوحيد: تقرير PDF يعرض النصوص العربية كنص بديل `[Arabic text - see project record]`
لأن خط Helvetica القياسي في PDF لا يدعم أحرف خارج Latin-1. لدعم عربي كامل في PDF مستقبلاً،
يجب إضافة خط TrueType عربي مضمّن (مثل Amiri) — البنية التحتية لذلك موجودة وجاهزة للتوسعة.

## كل الحسابات حقيقية وليست تقريبية

- طول القلبة المائلة للسلم = √(الطول الأفقي² + الارتفاع²) — نظرية فيثاغورس فعلية
- ضغط التربة = الحمل الفعلي ÷ مساحة القاعدة الفعلية
- الضغط الهيدروستاتيكي في الخزانات = γ_water × h (معادلة فيزيائية)
- وزن الحديد = (π/4 × قطر²) × كثافة الفولاذ × الطول الفعلي × العدد
- مركز ثقل القاعدة المشتركة يُحسب من مواقع وأحمال الأعمدة الفعلية
- عدد طوب الهوردي = مساحة البلاطة ÷ أبعاد الطوبة الفعلية
- طول التثبيت = معادلة ACI 318-19 الكاملة بمعاملات الموقع والطلاء وحجم السيخ ومقاومة الخرسانة الفعلية
- طول الخطاف = طول قوس الانحناء الفعلي (نصف قطر × الزاوية بالراديان) + الذيل المستقيم وفق الكود
- طول الكانة = محيط الشكل الصافي الفعلي (بعد طرح الغطاء الخرساني) + طولا الخطافين

جميع المعاملات (نسب الخلط، الأغطية الخرسانية، درجات الحديد، الأسعار، إلخ) قابلة للتعديل الكامل من واجهة المستخدم.

## حدود معروفة في هذا الإصدار

- حاسبة السلالم في القسم الثاني تُغطي حالياً مجرى واحد (Straight flight)؛ لأنواع L/U/الدائري استخدم القسم الأول مباشرة أو "عنصر مخصص"
- حاسبة البلاطة الهوردي في القسم الثاني تفترض قيماً افتراضية معقولة لسمك طوبة الهوردي عند عدم تحديدها يدوياً
- 17 نوع العنصر المطلوب بالكامل (خوازيق، قبعات، ميدات، جسور، أساسات شريطية منفصلة) مدعومة عبر "عنصر مخصص" (إدخال حر دقيق للقضبان والكانات) بدل نموذج مخصص لكل نوع، لضمان دقة المدخلات دون افتراضات هندسية غير مؤكدة لكل شكل

## تحديث: القسم الثالث - الجزء الثاني (طرق الاستيراد + الذكاء الاصطناعي)

تمت إضافة `backend/calculators/import/` وتشمل:

- **csvImporter.js**: parser حقيقي لملفات CSV (state machine على مستوى الأحرف)، يدعم الفواصل المختلفة، القيم المقتبسة، BOM، ويحول البيانات تلقائياً إلى بنود حصر (اسم، كمية، وحدة، سعر، تكلفة).
- **xlsxImporter.js**: parser حقيقي لملفات Excel (.xlsx) — يقرأ بنية ZIP يدوياً (Central Directory + Local Headers) ويفك ضغط DEFLATE عبر `zlib` المدمجة في Node.js، ثم يحلل XML الأوراق و sharedStrings دون أي مكتبة خارجية (لا `xlsx` ولا `exceljs`).
- **pdfImporter.js**: مستخرج نصوص حقيقي من PDF — يجد كائنات stream، يفك تشفير FlateDecode/ASCII85Decode/ASCIIHexDecode، ويحلل عمليات إظهار النص (`Tj`, `TJ`, `'`, `"`) لإعادة بناء الأسطر، مع محاولة استدلال جدول حصر كميات من النص.
- **dxfImporter.js**: محلل حقيقي لصيغة DXF النصية (Group Codes) يستخرج كيانات `LINE`, `LWPOLYLINE`, `CIRCLE`, `ARC`, `TEXT/MTEXT`, `DIMENSION` ويحسب منها الأطوال والمحيطات والمساحات الفعلية (وليس تقديرية) — بما في ذلك حساب المساحة بصيغة Shoelace للمضلعات المغلقة.
- **aiAnalyzer.js**: واجهة موحدة لاستدعاء Claude API (عبر `https` المدمجة، بدون SDK) لتحليل بيانات المخططات المستخرجة، مقارنة نسختين من حصر الكميات، والإجابة عن أسئلة هندسية حرة. تتطلب متغير البيئة `ANTHROPIC_API_KEY` على الخادم؛ في حال غيابه تعمل بقية الوظائف (CSV/Excel/PDF/DXF) بشكل كامل ومستقل.

### ملاحظات مهمة

- جميع الملفات الثنائية (xlsx, pdf) تُرسل كـ **Base64** ضمن جسم طلب JSON (`file_base64`) — لا حاجة لـ multipart لأن الخادم لا يستخدم أي مكتبات خارجية.
- تم رفع الحد الأقصى لحجم جسم الطلب إلى 30 ميجابايت لدعم الملفات المشفّرة Base64.
- **IFC (BIM)** والحصر التفاعلي المباشر من المخططات سيُضافان في تحديث لاحق (الجزء الثالث: الواجهة والتكامل الكامل)، لأنهما يتطلبان محللات أكبر (IFC STEP format) وواجهة رسم تفاعلية.
- جميع الحسابات (الأطوال، المساحات، المحيطات) في `dxfImporter.js` مبنية على الإحداثيات الفعلية المستخرجة من الملف، وليست معاملات تقريبية — وفق طلب المستخدم الصريح.

## تحديث: القسم الثالث - الجزء الثالث (مكتبة الأسعار المركزية + التقارير)

تمت إضافة `backend/utils/priceLibrary.js`, `backend/utils/boqReports.js`, `backend/utils/tablePdfGenerator.js`, `backend/utils/xlsxWriter.js`, `backend/utils/csvWriter.js`.

### مكتبة الأسعار المركزية (`priceLibrary.js`)

قاعدة بيانات أسعار حقيقية مخزَّنة على القرص (`backend/data/price_library.json` — يُنشأ تلقائياً عند أول تشغيل)، بلا أي تبعيات خارجية:

- **أسعار المواد**: كل بند من كل حاسبة BOQ (طوب، بلوك، سيراميك، رخام، دهانات، عزل، كهرباء، صحي، طرق، تربة...) + الخرسانة والحديد.
- **أسعار العمالة، المعدات، النقل، الضرائب، الخصومات.**
- **الموردين**: إضافة/بحث/حذف مورد لكل بند، مع تحديد أرخص مورد تلقائياً (`findCheapestSupplier`).
- **تعدد المستويات**: سعر عام (Global) يمكن تجاوزه بسعر خاص بمنطقة (Region) يمكن تجاوزه بسعر خاص بمشروع (Project). القراءة تدمج الثلاثة بالأولوية الصحيحة (`getEffectivePrices`) دون فقدان القيم الافتراضية غير المُعدَّلة.
- **تسعير بند واحد** (`priceLineItem`) يُستخدم من وحدة التقارير لتسعير أي بند حصر كميات تلقائياً من المكتبة، أو بسعر مباشر يتجاوزها.

### وحدة التقارير (`boqReports.js`)

بنية **بند حصر موحّد (Normalized Line Item)** تختزل مخرجات أي حاسبة BOQ (لكل منها شكل مختلف) إلى شكل واحد قابل للتجميع والتسعير والتصدير. تنتج 8 أنواع تقارير مطلوبة في المستند:

1. **تقرير الكميات** - تجميع حسب القسم والوحدة.
2. **جدول الكميات (BOQ)** - كل بند مسعّر + الإجمالي الكلي.
3. **تقرير الأسعار** - مصدر كل سعر (مكتبة/مباشر/غير مسعّر) والبنود غير المسعّرة.
4. **تقرير التكاليف** - مواد + عمالة + معدات + نقل، خصم، ضريبة، إجمالي نهائي، وتوزيع حسب القسم.
5. **تقرير الهدر** - كمية الهدر الفعلية لكل بند ومتوسط نسبة الهدر.
6. **تقرير مقارنة الكميات** (وأساس تقرير الفروقات بين المخططات، بالدمج مع `aiAnalyzer.compareBOQVersions`) - إضافة/حذف/تعديل بين نسختين، بالنسبة والفرق الفعلي.
7. **تقرير الإنجاز** - نسبة تنفيذ كل بند (منفَّذ ÷ مخطط) وإجمالي نسبة إنجاز المشروع.
8. **تقرير ملخص المشروع** - يجمع كل التقارير أعلاه في نظرة عامة واحدة.

### التصدير الحقيقي (بدون أي مكتبة خارجية)

- **PDF احترافي بجداول حقيقية** (`tablePdfGenerator.js`): يبني على نفس أسلوب `nativePdfGenerator.js` (كتابة PDF 1.4 خام) مع طبقة رسم جداول كاملة (ترويسة ملونة، تلوين صفوف متبادل، حدود خلايا، تقسيم صفحات تلقائي A4 أفقي مع تكرار الترويسة). تم التحقق من صحة الملفات الناتجة عبر `pypdf` (صفحة واحدة لـ 2 بند، 3 صفحات لـ 60 بند تلقائياً).
- **Excel حقيقي (.xlsx)** (`xlsxWriter.js`): يبني أرشيف ZIP يدوياً (Local File Header + Central Directory + EOCD مع CRC32 مطابق للمعيار) ويضغط بـ `zlib.deflateRawSync` المدمجة في Node.js - نفس الأسلوب المستخدم فعلياً في قراءة xlsx بالمشروع (`xlsxImporter.js`) لكن بالاتجاه المعاكس. تم التحقق من صحة الملف عبر فتحه بـ Python `zipfile` وقراءة XML الورقة.
- **CSV** (`csvWriter.js`): متوافق RFC 4180 مع BOM لعرض عربي سليم في Excel.
- **طباعة مباشرة**: صفحة HTML عربية RTL جاهزة بزر طباعة (`window.print()`) تُحفظ في `/reports/` وتُفتح مباشرة من المتصفح.

جميع مسارات `/api/reports/*` تستقبل `items: [{category, description, quantity, unit, wastePercent, priceKey, unitPrice}]` (البند الموحّد)، بحيث يمكن تغذيتها مباشرة من مخرجات أي حاسبة BOQ موجودة (`/api/boq/*`) بعد تحويلها لهذا الشكل.

### ملاحظة تقنية

مكتبة الأسعار مخزَّنة حالياً في ملف JSON على القرص (قاعدة بيانات مركزية بسيطة بلا تبعيات، متوافقة مع سياسة "صفر تبعيات" للمشروع بالكامل). عند الانتقال لاحقاً لقاعدة بيانات حقيقية (PostgreSQL/MongoDB) في مرحلة التكامل الشامل، الواجهة البرمجية لـ `priceLibrary.js` (الدوال المُصدَّرة) تبقى كما هي دون تغيير في بقية النظام.

### الأجزاء المتبقية من القسم الثالث (لتحديث لاحق)

- الواجهة الأمامية الكاملة لعرض/إدخال جميع نماذج BOQ الـ 60+ وربطها بهذه الـ APIs (`frontend/`).
- لوحة التحكم الرئيسية (Dashboard) بالإحصائيات المباشرة المذكورة في المستند.
- عارض المخططات التفاعلي (PDF/DWG) مع القياس والتلوين المباشر على المخطط.
- استيراد/تحليل IFC (BIM).
- التكامل الكامل مع إدارة المشاريع، الميزانية، المستندات، الجودة الوارد ذكرها في القسم الأول.
