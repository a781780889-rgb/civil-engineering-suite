# حالة القسم الثاني عشر: نظام إدارة المخططات الهندسية (Drawings Management)

## ما تم إنجازه في هذه الجلسة: الجزء 10ج/10 (الأخير) — التكامل الشامل مع بقية أقسام النظام

هذا هو الجزء الأخير المتبقي من القسم الثاني عشر بالكامل. تم تنفيذه كوظائف حقيقية
تعمل فعلياً (وليس واجهات شكلية)، بنفس النمط المعماري المُثبَت فعلياً في
`documentModuleLinks.js` (القسم الحادي عشر، البند المتبقي — التكامل مع باقي الأقسام).

### 1) طبقة ربط عامة جديدة: `backend/utils/drawingModuleLinks.js`

جدول ربط عام (`drawing_id <-> module/entity`) يخزَّن في
`backend/data/drawing_module_links.json`، يدعم 10 أقسام مطابقة حرفياً لبند
"التكامل مع الأقسام الأخرى" الوارد في مواصفة القسم الثاني عشر:

| المفتاح (module) | التسمية |
|---|---|
| `project` | إدارة المشاريع |
| `survey` | تطبيق المساحة |
| `boq` | حصر الكميات |
| `quality` | إدارة الجودة |
| `safety` | السلامة المهنية |
| `documents` | إدارة المستندات |
| `business` | إدارة الأعمال |
| `equipment` | إدارة المعدات |
| `schedule` | الجدول الزمني |
| `reports` | نظام التقارير |

الدوال المُصدَّرة:

| الدالة | الوصف |
|---|---|
| `linkDrawing` | ربط مخطط موجود فعلياً بسجل في أي قسم مدعوم، مع منع تكرار نفس الربط |
| `linkDrawings` | ربط عدة مخططات دفعة واحدة بنفس السجل |
| `unlinkDrawing` | فك ربط، مع سجل تدقيق كامل (Audit Log) |
| `getLinkedDrawings` | جلب كل المخططات المرتبطة بسجل معيّن داخل قسم معيّن — هذه هي الدالة التي يستدعيها أي قسم آخر لعرض "المخططات المرتبطة" مباشرة في واجهته |
| `getLinkedDrawingsByProject` | جلب كل الروابط الخاصة بمشروع كامل، مجمّعة حسب القسم — لعرضها في لوحة تحكم المشروع |
| `getModulesLinkedToDrawing` | جلب كل الأقسام التي يظهر فيها مخطط معيّن — مفيد داخل شاشة المخطط نفسه |
| `getSupportedModules` | القائمة المرجعية للأقسام المدعومة (للواجهات) |
| `getIntegrationSummary` | ملخص إحصائي عام لعدد الروابط حسب كل قسم |
| `linkDrawingToDocument` | ربط اختياري بمعرّف مستند في نظام إدارة المستندات (DMS، القسم الحادي عشر)، بحيث يمكن إدراج المخطط كمرجع مستندي موحّد ضمن ملف المشروع |

**تحقق فعلي عبر استدعاء الـ API handlers الحقيقية (end-to-end، وليس اختبار وحدة
معزول):** تسجيل دخول فعلي بتوكن جلسة حقيقي → إنشاء مشروع ومخطط حقيقيين → ربط
المخطط بالمشروع → ربط دفعي بقسم الجودة → التحقق من منع تكرار الربط → استعلام
المخططات المرتبطة بالمشروع، وبالعكس الأقسام المرتبطة بالمخطط → فك الربط → التحقق
من رفض أي طلب بتوكن غير صالح (فحص أمني). جميع الخطوات التسعة اجتازت الاختبار
بنجاح تام قبل التسليم.

### 2) 9 مسارات API جديدة في `server.js` تحت `/api/drawings/links/*`

جميعها محمية بنفس نظام الصلاحيات (`requirePermission(req, 'drawings', ...)`)
المستخدم في بقية أجزاء القسم:

- `GET  /api/drawings/links/modules`
- `POST /api/drawings/links/link`
- `POST /api/drawings/links/link-many`
- `POST /api/drawings/links/unlink`
- `GET  /api/drawings/links/for-entity`
- `GET  /api/drawings/links/by-project`
- `GET  /api/drawings/links/for-drawing`
- `GET  /api/drawings/links/summary`
- `POST /api/drawings/links/link-document`

تم التحقق من تحميل السيرفر بالكامل (969 مسار إجمالي بعد الإضافة، من ضمنها 118
مسار خاص بالمخططات) دون أي خطأ صياغة أو تحميل.

### 3) عنصر واجهة عام: `backend/frontend/js/drawingModuleLinks.js`

ودجت جاهز لإعادة الاستخدام من أي قسم آخر عبر استدعاء واحد:

```js
drawLinksRender({
  containerId: 'my-container-id',
  module: 'quality',      // أو أي قسم من الأقسام العشرة المدعومة
  entityId: someRecordId,
  projectId: currentProjectId,
});
```

يوفر أيضاً `drawLinksRenderProjectSummary` لعرض ملخص عدد المخططات المرتبطة بكل
قسم داخل لوحة تحكم مشروع كامل.

### 4) تكامل فعلي مُدمَج ومختبَر: تبويب "المستندات" في تفاصيل المشروع (`pm.js`)

تمت إضافة قسم "المخططات الهندسية المرتبطة" فعلياً داخل `pmRenderDocuments` في
قسم إدارة المشاريع (القسم الرابع)، والذي يستدعي `drawLinksRender` تلقائياً عند
فتح تبويب مستندات أي مشروع — فيعرض كل المخططات الهندسية المرتبطة بذلك المشروع
تحديداً مباشرة، مع إمكانية ربط مخطط جديد أو فك ربط موجود من نفس الشاشة، دون أي
تعديل على منطق المستندات الداخلي الموجود مسبقاً في نفس التبويب.

**ملاحظة نطاق العمل:** بقية الأقسام المستهدفة (BOQ، الجودة، السلامة، المعدات،
المساحة، الجدول الزمني) لا تملك حالياً بنية تبويبات تفصيلية مكافئة لما هو موجود
في `pm.js` (subtabs) تسمح بحقن آمن دون إعادة هيكلة واجهاتها. الودجت العام
(`drawLinksRender`) جاهز ومختبَر ويعمل فوراً بمجرد استدعائه بثلاث معطيات فقط
(`containerId`, `module`, `entityId`) من داخل أي شاشة تفصيل سجل في تلك الأقسام
عند تطويرها مستقبلاً، بنفس الطريقة المُستخدَمة فعلياً الآن في `pm.js`.
